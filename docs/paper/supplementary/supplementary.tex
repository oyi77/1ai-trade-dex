\section{Extended Proofs and Bounds}
\label{sec:proofs}

\subsection{Promotion Pipeline Prevents Catastrophic Deployment}

\begin{proposition}
If the promotion pipeline requires minimum Sharpe ratio $S_{\min} > 0$ for PAPER$\to$LIVE transition, then no strategy with negative drift can be promoted to live trading.
\end{proposition}

\begin{proof}
By definition, the Sharpe ratio is:
\begin{equation}
S = \frac{\mathbb{E}[R] - R_f}{\sigma_R}
\end{equation}
where $R$ is the strategy return, $R_f$ is the risk-free rate, and $\sigma_R$ is the standard deviation of returns.

If $S > 0$, then $\mathbb{E}[R] > R_f$, which means the strategy has positive expected excess return. A strategy with negative drift has $\mathbb{E}[R] < 0 < R_f$, so $S < 0$. The gate $S \geq S_{\min} > 0$ therefore excludes all negative-drift strategies from LIVE promotion.
\end{proof}

\begin{corollary}
The PolyEdge promotion pipeline with thresholds $\text{MIN\_SHARPE\_PAPER} = 0.5$, $\text{MIN\_WIN\_RATE} = 0.45$, and $\text{MAX\_DRAWDOWN} = 0.20$ enforces a minimum risk-adjusted return at least $0.5\sigma_R$ above the risk-free rate, while capping maximum drawdown at 20\%.
\end{corollary}

\subsection{Kelly Criterion for Binary Prediction Markets}

\begin{proposition}
For a binary prediction market with contract price $p$ (reflecting implied probability) and true probability $\pi$, the Kelly-optimal bet size is:
\begin{equation}
f^* = \frac{\pi(1-p) - (1-\pi)p}{p(1-p)}
\end{equation}
\end{proposition}

\begin{proof}
In a binary market, buying YES at price $p$ yields payoff $\$1$ with probability $\pi$ and $\$0$ with probability $1-\pi$. The expected log-growth is:
\begin{equation}
G(f) = \pi \ln(1 + f/p - f) + (1-\pi)\ln(1 - f)
\end{equation}

Maximizing with respect to $f$:
\begin{equation}
\frac{dG}{df} = \frac{\pi(1-p)}{p + f(1-p)} - \frac{1-\pi}{1-f} = 0
\end{equation}

Solving for $f$ yields $f^* = \frac{\pi - p}{p(1-p)} \cdot p = \frac{\pi(1-p) - (1-\pi)p}{p(1-p)}$.
\end{proof}

\textbf{Practical implementation}: PolyEdge uses fractional Kelly ($f = 0.25 \cdot f^*$ for the ``safe'' risk profile and $f = 0.50 \cdot f^*$ for ``normal'') to reduce variance at the cost of expected growth rate.

\subsection{Thompson Sampling Allocation}

The bankroll allocator rank-strategies using Thompson sampling from a Beta distribution:
\begin{equation}
S_k \sim \text{Beta}(\alpha_k + 1, \beta_k + 1)
\end{equation}
where $\alpha_k$ is the cumulative win count and $\beta_k$ is the cumulative loss count for strategy $k$. This naturally balances exploration (trying under-sampled strategies) with exploitation (allocating to high-performing strategies).

\subsection{Autonomous Promotion Gate}

The core promotion decision logic from \texttt{autonomous\_promoter.py}, showing the health-based evaluation criteria for PAPER$\to$LIVE transition:

\begin{lstlisting}[caption={Paper-to-Live promotion criteria (autonomous\_promoter.py, lines 80--110)}]
def _check_paper_criteria_from_health(
    self, exp: ExperimentRecord, health: dict
) -> tuple[bool, list[str]]:
    """Evaluate paper->live promotion using current health metrics."""
    reasons = []
    trades = health.get("total_trades", 0)
    win_rate = health.get("win_rate", 0.0)
    sharpe = health.get("sharpe", 0.0)
    max_dd = health.get("max_drawdown", 0.0)

    if trades < self.MIN_TRADES_PAPER:
        reasons.append(f"trades {trades} < {self.MIN_TRADES_PAPER}")
    if win_rate < self.MIN_WIN_RATE_PAPER:
        reasons.append(f"win_rate {win_rate:.1%} < "
                       f"{self.MIN_WIN_RATE_PAPER:.1%}")
    if sharpe < self.MIN_SHARPE_PAPER:
        reasons.append(f"sharpe {sharpe:.2f} < "
                       f"{self.MIN_SHARPE_PAPER:.2f}")
    if max_dd > self.MAX_DRAWDOWN_PAPER:
        reasons.append(f"dd {max_dd:.1%} > "
                       f"{self.MAX_DRAWDOWN_PAPER:.1%}")

    passed = len(reasons) == 0
    return passed, reasons
\end{lstlisting}

The \texttt{ExperimentRecord} transitions through stages DRAFT $\to$ SHADOW $\to$ PAPER $\to$ LIVE\_PROMOTED $\to$ RETIRED. Each transition requires satisfying all criteria; any single failure blocks promotion.

\subsection{Local Debate Fallback Engine}

When the external MiroFish debate service is unavailable, PolyEdge falls back to a local debate engine that evaluates signals using heuristic analysis:

\begin{lstlisting}[caption={Local debate fallback (services/mirofish\_service.py, adapted)}]
class LocalDebateEngine:
    """Fallback debate engine when MiroFish is unavailable.
    Uses rule-based bull/bear synthesis for basic signal validation.
    """
    def evaluate_signal(self, signal: TradingSignal) -> DebateVerdict:
        bull_args = self._bull_case(signal)
        bear_args = self._bear_case(signal)
        verdict = self._adjudicate(bull_args, bear_args, signal)
        return verdict

    def _bull_case(self, signal: TradingSignal) -> list[str]:
        args = []
        if signal.confidence and signal.confidence >= 0.7:
            args.append(f"High confidence ({signal.confidence:.0%})")
        if signal.edge_pct and signal.edge_pct >= 0.05:
            args.append(f"Edge detected ({signal.edge_pct:.1%})")
        if self._trending_favorable(signal):
            args.append("Market trend favorable")
        return args

    def _bear_case(self, signal: TradingSignal) -> list[str]:
        args = []
        if signal.confidence and signal.confidence < 0.5:
            args.append(f"Low confidence ({signal.confidence:.0%})")
        if signal.edge_pct and signal.edge_pct < 0.03:
            args.append(f"Insufficient edge ({signal.edge_pct:.1%})")
        if self._counter_indicator(signal):
            args.append("Counter-indicator present")
        return args

    def _adjudicate(self, bull, bear, signal) -> DebateVerdict:
        if len(bull) > len(bear) and signal.confidence and signal.confidence >= 0.6:
            return DebateVerdict(should_trade=True, confidence=0.65)
        return DebateVerdict(should_trade=False, confidence=0.35)
\end{lstlisting}

\subsection{Fitness Score Computation}

The composite fitness function that drives evolutionary selection:

\begin{lstlisting}[caption={Fitness calculation (domain/evolution/fitness.py)}]
def calculate_fitness(metrics: FitnessMetrics) -> float:
    """Calculate fitness score from 0.0 (worst) to 1.0 (best).
    Requires at least 20 trades for meaningful evaluation."""
    if metrics.total_trades < 20:
        return 0.0

    score = (
        (normalize(metrics.sharpe_ratio, -3, 3) * 0.30) +
        (metrics.win_rate * 0.20) +
        (normalize(metrics.profit_factor, 0, 5) * 0.15) +
        ((1.0 - metrics.max_drawdown_pct) * 0.15) +
        (normalize(metrics.alpha_per_trade, -1, 1) * 0.10) +
        (metrics.capital_rotation_efficiency * 0.10)
    )
    return max(0.0, min(1.0, score))
\end{lstlisting}

The weight allocation (30\% Sharpe, 20\% win rate, 15\% profit factor, 15\% drawdown penalty, 10\% alpha, 10\% capital rotation) was chosen to prioritize risk-adjusted returns while maintaining diversity pressure through the capital rotation efficiency term.

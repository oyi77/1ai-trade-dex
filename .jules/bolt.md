## 2023-11-20 - [TradesTable filter optimization]
**Learning:** `TradesTable.tsx` originally contained four separate `.filter()` operations on the trades array for the UI filters within every render cycle (or rather they were computed dynamically before the return on every render). By memoizing the counts and resolving them in a single `O(N)` loop we avoid redundant full array traversals (`O(N*4)`) during render.
**Action:** When working with large datasets in UI components, always look for opportunities to consolidate multiple `.filter()` or `.map()` operations over the same array into a single pass and wrap it in `useMemo` so it's not triggered unnecessarily by component re-renders. Additionally, ensure all hooks (like `useMemo`) are called before any early returns to prevent React Hook order violations.

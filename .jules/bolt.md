## 2025-06-08 - [React Array Spread Optimization]
**Learning:** Using `Math.max(...array)` or `Math.min(...array)` with the spread operator on mapped arrays created from backend data is an anti-pattern in this codebase's React components. It creates intermediate array allocations that trigger garbage collection, but more dangerously, it risks throwing "Maximum call stack size exceeded" errors if the `array` length exceeds the JavaScript engine's arguments limit (typically ~65k).
**Action:** Always replace spread-based min/max aggregations with `Array.prototype.reduce()` or single-pass `for...of` loops, and wrap the results in `useMemo` hooks. When replacing `Math.max(0, ...array)` with `.reduce()`, explicitly set `0` as the initial value to preserve chart scaling boundaries. If no explicit `0` floor exists, initialize with `-Infinity` to avoid skewing negative values.

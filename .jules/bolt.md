## 2024-06-02 - [TradesTable Filter Counts O(N) Loop Memoization]
**Learning:** Found an anti-pattern in the React components where derived counts were calculated directly in the render function body via multiple `filter(..).length` calls on large arrays instead of using a single `O(N)` loop wrapped in `useMemo`. When optimizing React components, replacing chained high-order array methods (like `filter`, `map`, `reduce`) with single-pass iterative `for` loops inside `useMemo` hooks can significantly reduce blocking times on the main UI thread during renders.
**Action:** Always scan for unmemoized multi-pass derived array calculations within React render functions, especially in large dashboard or list views. Consolidate them into single `for` loop passes wrapped with `useMemo`.
